OWL Change Ontology: Description back to ToC

An ontology for describing changes between OWL ontology versions

Instantiating the OWL Change Ontology back to ToC

This is an example of how to instantiate the OWL Change Ontology. It shows the use of the different change operations from the ontology. First of all the example shows the changelog that contains the ontology changes (ex:log234), and information on the ontology versions it covers (och:prevVersion, and och:newVersion).
Turtle Syntax
    ex:log234 a och:ChangeLog ;
    och:prevVersion epo:3.1.0;
    och:newVersion epo:4.0.0.
The following example show an entity change, the addition of a class, along with term specific metadata. First the URI corresponding to the change is instantiated as an och:AddClass change operation, then it is indicated what log it belongs to (och:fromChangelog), who issued said change (och:issuedBy), the date it was issued (dct:issued) and the term that has been added, in this instance a class (och:addedClass).
Turtle Syntax
    ex:ac1 a och:AddClass ;
    och:fromChangelog ex:log234 ;
    och:issuedBy ex:Jane_Doe ;
    dct:issued "2023-11-24"^^xsd:date ;
    och:addedClass ex:Class1 .
The next example show an entity being renamed. The URI is instantiated as an och:RenameTerm change operation, then the metadata is used to indicate provenance, author,date it was issued. The name that the OWL entity used to have (och:outdatedEntityName), and the term that it has been renamed to (och:renamedEntityName) are indicated.
Turtle Syntax
    ex:rt1 a och:RenameTerm ;
    och:fromChangelog ex:log234 ;
    och:issuedBy ex:Jane_Doe ;
    dct:issued "2023-11-24"^^xsd:date ;
    och:outdatedEntityName ex:Identifier ;
    och:renamedEntityName ex:Id .
The following is an example of the addition of a property. Besides the aforementioned provenance, author, and date, there is a property for the object property added in that change (och:addedObjectProperty), and related changes for adding the range and domain of said property (och:relatedChange). Within the domain and range addition operations there are properties indicating the related property (och:addedDomainToProperty, och:addedRangeToProperty), and properties pointing to the terms domain and range that are added (och:addedDomain, och:addedRange).
Turtle Syntax
    ex:aop1 a och:AddObjectProperty ;
        och:fromChangelog ex:log234 ;
        och:issuedBy ex:Jane_Doe ;
        dct:issued "2023-09-25"^^xsd:date ;
        och:addedObjectProperty ex:exampleObjectProperty .
    ex:ad1 a och:AddDomain ;
        och:fromChangelog ex:log234 ;
        och:issuedBy ex:Jane_Doe ;
        dct:issued "2023-09-25"^^xsd:date ;
        och:addedDomainToProperty ex:exampleObejctProperty ;
        och:addedDomain ex:exampleDomain :
        och:relatedChange ex:aop1 .
    ex:ar1 a och:AddRange ;
        och:fromChangelog ex:log234 ;
        och:issuedBy ex:Jane_Doe ;
        dct:issued "2023-09-25"^^xsd:date ;
        och:addedRangeToProperty ex:exampleObjectProperty ;
        och:addedRange ex:exampleRange ;
        och:relatedChange ex:aop1 .
The last example is one for removing a Characteristic from a property. Besides the aforementioned provenance, author, and date, there is a property for the property that the characteristic belongs to (och:removedCharacteristicFromProperty). Then there is a property to indicate the property that has been added (och:removedCharacteristic), in this case och:SymmetricProperty.
Turtle Syntax
    ex:rc1 a och:RemoveCharacteristic ;
        och:fromChangelog ex:log234 ;
        och:issuedBy ex:Jane_Doe ;
        dct:issued "2023-09-25"^^xsd:date ;
        och:removedCharacteristicFromProperty ex:exampleObjectProperty;
        och:removedCharacteristic och:SymmetricProperty .
